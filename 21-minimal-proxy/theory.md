# День 21: minimal proxy EIP 1167

## в чем проблема обычных контрактов

Если тебе нужно создать много одинаковых контрактов  

каждый деплой стоит газ  

и чем сложнее логика  
тем дороже  

Пример  

ты хочешь создать 100 кошельков  
или 100 пулов  

деплоить полный контракт каждый раз дорого  

## решение

Minimal proxy  

Это очень маленький контракт  
который не содержит логики  

он просто перенаправляет все вызовы  

## как это работает

есть один основной контракт  
с полной логикой  

и есть прокси  

прокси получает вызов  
и через delegatecall отправляет его в основной контракт  

код выполняется там  
но данные хранятся в прокси  

## почему это дешево

прокси контракт очень маленький  

его деплой стоит мало газа  

логика не копируется каждый раз  

## простой пример

есть контракт кошелька  

ты создаёшь один основной контракт  

потом создаёшь много прокси  

каждый прокси ведёт себя как полноценный контракт  
но стоит дешевле  

## что такое EIP 1167

это стандарт minimal proxy  

он описывает как должен выглядеть такой прокси  

чтобы он был максимально простой и дешёвый  

## что происходит внутри

прокси хранит адрес основного контракта  

при вызове  

он делает delegatecall  

и выполняет код  

## важный момент

у каждого прокси свой storage  

поэтому  

у каждого пользователя свои данные  

хотя логика одна  

## когда используется

кошельки  
DeFi протоколы  
factory системы  

## пример из жизни

представь что у тебя есть шаблон  

ты не создаёшь копию всего  

ты просто используешь один шаблон  
для всех  

## главная мысль

minimal proxy экономит газ  

логика хранится в одном месте  

прокси только перенаправляет вызовы  

это позволяет масштабировать систему дешево и эффективно
