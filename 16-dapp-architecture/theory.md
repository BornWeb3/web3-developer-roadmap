# День 16: реальная dapp архитектура

## почему одного контракта недостаточно

В начале кажется что достаточно написать контракт  
задеплоить его  
и приложение готово  

Но это не так  

Контракт это только часть системы  

Реальное dapp это набор компонентов  
которые работают вместе  

## из чего состоит dapp

Есть несколько основных частей  

смарт контракты  
фронтенд  
кошелек пользователя  
RPC  
индексаторы  

Все они связаны между собой  

## как это работает в целом

Пользователь открывает сайт  

подключает кошелек  

интерфейс получает доступ к его адресу  

дальше интерфейс может  

читать данные  
или отправлять транзакции  

## роль фронтенда

Фронтенд это обычное приложение  

оно отвечает за  

кнопки  
страницы  
отображение данных  

Сам фронтенд не знает блокчейн напрямую  

он работает через библиотеки и RPC  

## роль кошелька

Кошелек это мост между пользователем и блокчейном  

он  

подписывает транзакции  
хранит приватный ключ  

Фронтенд не может отправить транзакцию без кошелька  

## роль RPC

RPC это доступ к ноде  

через него фронтенд  

читает данные  
отправляет запросы  

Без RPC приложение не сможет общаться с сетью  

## роль смарт контрактов

Контракты это логика  

они  

хранят состояние  
выполняют код  
обрабатывают транзакции  

Все важные действия происходят именно здесь  

## роль индексаторов

Индексаторы читают events  

и строят удобные базы данных  

Это нужно чтобы быстро получать данные  

Пример  

история транзакций  
список действий пользователя  

## простой пример

Ты заходишь на dapp  

подключаешь кошелек  

нажимаешь кнопку swap  

фронтенд формирует транзакцию  

кошелек подписывает её  

RPC отправляет в сеть  

контракт выполняет swap  

event записывается  

фронтенд обновляет данные  

## почему это важно

если понимать архитектуру  

становится понятно  

где логика  
где данные  
где происходят ошибки  

## пример из жизни

dapp это как обычное приложение  

фронтенд это интерфейс  
контракт это сервер  
RPC это канал связи  
кошелек это авторизация  

## главная мысль

dapp это не один контракт  

это система из нескольких частей  

если понимаешь архитектуру  
ты начинаешь видеть как всё работает вместе
