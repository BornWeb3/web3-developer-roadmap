# День 61: deploy pipeline и env

## почему это важно

Deploy это не просто нажать кнопку  

в реальных проектах  

есть целый процесс  

подготовка  
настройка  
проверки  
и только потом deploy  

## что такое deploy pipeline

Это последовательность действий  

которая помогает  

безопасно и удобно  
выкатывать контракты  

## что входит в pipeline

компиляция  

тесты  

проверка env  

deploy  

verify  

## почему это важно

если делать deploy вручную  

легко ошибиться  

не тот RPC  
не тот private key  
не та сеть  

## что такое env

Env это переменные окружения  

секретные или настраиваемые данные  

## пример

RPC URL  

private key  

адреса контрактов  

## почему нельзя хранить это в коде

если случайно залить private key в GitHub  

можно потерять все средства  

## как это работает

данные хранятся отдельно  

а код читает их во время запуска  

## простой пример

скрипт deploy  

берёт  

RPC  
private key  
chain id  

из env файла  

## важный момент

для разных сетей  

обычно разные env  

## пример

mainnet env  

и отдельно testnet env  

## что ещё важно

автоматизация  

чем меньше ручных действий  

тем меньше шанс ошибки  

## пример из жизни

pipeline это как чеклист перед запуском ракеты  

всё идёт по шагам  

## главная мысль

deploy pipeline делает deploy безопаснее  

env хранит важные настройки отдельно  

это помогает избежать ошибок  
и защищает чувствительные данные
