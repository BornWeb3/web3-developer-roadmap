# День 14: gas оптимизация и storage стратегии

## почему это важно

Каждая операция в Ethereum стоит газ  

Чем сложнее код  
тем дороже транзакция  

Если писать не оптимально  
пользователь будет переплачивать  

В DeFi это критично  

## откуда берётся расход газа

Самое дорогое это запись в storage  

создать новую переменную  
изменить значение  

это стоит больше всего  

Чтение дешевле  
но тоже влияет на стоимость  

## простая логика

меньше записей в storage  
меньше газа  

меньше операций  
меньше газа  

## пример

если ты записываешь значение в storage несколько раз  
ты платишь каждый раз  

лучше сначала посчитать всё  
и записать один раз  

## storage стратегии

Важно как ты хранишь данные  

EVM использует ячейки по 32 байта  

если использовать их правильно  
можно экономить газ  

## packing

маленькие переменные можно упаковать в одну ячейку  

Пример  

uint128 и uint128  
могут лежать вместе  

это уменьшает количество используемых слотов  

## порядок переменных

порядок влияет на упаковку  

если поставить большие переменные между маленькими  
упаковка не сработает  

лучше группировать одинаковые типы  

## mapping vs массивы

mapping дешевле для поиска  

массивы удобны для перебора  
но могут быть дорогими  

нужно выбирать в зависимости от задачи  

## memory и calldata

memory используется внутри функций  

calldata используется для входных данных  

calldata дешевле чем memory  
потому что не копирует данные  

## пример

если функция принимает массив  
лучше использовать calldata  

это уменьшит расход газа  

## лишние операции

каждая операция стоит газ  

если можно упростить логику  
нужно это делать  

Пример  

не делать лишние вычисления  
не вызывать функции без необходимости  

## пример из жизни

представь что газ это деньги  

каждая строка кода это трата  

чем проще код  
тем дешевле выполнение  

## главная мысль

самое дорогое это storage  

правильная структура данных экономит газ  

простая логика уменьшает расходы  

если думаешь о газе  
ты пишешь более эффективные контракты
