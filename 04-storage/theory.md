# День 4: storage глубже и как EVM хранит данные

## что такое storage

Storage это место, где смарт контракт хранит свои данные.  

Это постоянная память. Всё что записано туда остаётся в блокчейне навсегда.  

Пример  

у тебя есть контракт токена  
в storage хранятся балансы пользователей  

если ты отправил токены  
баланс меняется и сохраняется  

## почему это важно

Storage это самая дорогая часть в Ethereum  

Любая запись стоит газ  
чтение стоит дешевле  
но тоже не бесплатно внутри транзакций  

Поэтому важно понимать как всё устроено  

## как EVM хранит данные

EVM делит storage на ячейки  

каждая ячейка это 32 байта  

и у каждой есть свой номер  

Пример  

можно представить storage как таблицу  

ячейка 0  
ячейка 1  
ячейка 2  

в каждой лежит значение  

## простой пример

```solidity
uint256 public count = 10;
```
это значение будет храниться в одной ячейке

EVM положит число 10 в storage

## как хранятся переменные

Переменные кладутся последовательно

```solidity
uint256 a;
uint256 b;
```
a будет в одной ячейке
b в следующей

## packing оптимизация

Если переменные маленькие
EVM может упаковать их в одну ячейку

Пример
```solidity
uint128 a;
uint128 b;
```
оба значения могут лежать в одной ячейке

это экономит газ

## mapping и сложные структуры

mapping хранится не подряд

используется hash

Пример
```solidity
mapping(address => uint256) balances;
```
значение не лежит просто в ячейке

EVM делает hash от ключа и слота
и по этому адресу хранит данные

Это нужно чтобы быстро находить значения

## простой пример

у тебя есть mapping балансов

когда ты запрашиваешь баланс
контракт вычисляет где он лежит
и возвращает значение

## разница между memory и storage

storage это постоянные данные
memory это временные данные

memory используется внутри функции
и исчезает после выполнения

Пример

ты считаешь значение внутри функции
оно хранится в memory
и потом исчезает

## почему это важно на практике

если неправильно работать со storage

ты будешь тратить больше газа
или можешь сломать логику

Пример

лишняя запись в storage
лишние расходы

## главная мысль

Storage это постоянная память контракта

EVM хранит данные в ячейках по 32 байта

простые переменные кладутся подряд
mapping хранится через hash

если понимаешь storage
ты начинаешь писать более дешёвые и правильные контракты
