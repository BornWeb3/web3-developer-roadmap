# День 44: protocol architecture, соединяем все компоненты

## почему это важно

Отдельные контракты это ещё не протокол  

Настоящий DeFi проект  
это система  

где много компонентов работают вместе  

## что входит в архитектуру

vault  
staking  
governance  
oracle  
liquidity pool  
treasury  
reward system  

всё связано между собой  

## как это выглядит

пользователь взаимодействует с фронтендом  

фронтенд работает с контрактами  

контракты используют  

oracle  
vault  
governance  
и другие модули  

## главная идея

каждый контракт отвечает за свою задачу  

не нужно делать всё в одном месте  

## пример

staking контракт  

не должен хранить логику governance  

oracle не должен заниматься наградами  

## почему разделение важно

код становится  

проще  
безопаснее  
удобнее для обновлений  

## как компоненты общаются

контракты вызывают друг друга  

передают данные  
используют интерфейсы  

## простой пример

vault хранит средства  

strategy зарабатывает доход  

oracle даёт цену  

governance меняет параметры  

## важный момент

архитектура должна быть продумана заранее  

если всё связано хаотично  

система станет сложной  
и опасной  

## частые проблемы

слишком много логики в одном контракте  

жёсткая зависимость компонентов  

неправильные права доступа  

## подход

делать маленькие модули  

каждый отвечает за одну задачу  

## пример из жизни

это как команда  

у каждого своя роль  

но вместе они создают систему  

## главная мысль

protocol architecture это структура всего проекта  

контракты разделяются по задачам  

и работают вместе как единая система  

именно так строятся реальные DeFi протоколы
