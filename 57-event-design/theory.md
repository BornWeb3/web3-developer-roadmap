# День 57: event design для indexing

## почему events важны

Смарт контракт хранит состояние  

но фронтенд  
аналитика  
и indexer системы  

чаще всего работают именно через events  

## что такое indexing

Indexing это процесс  

когда данные из блокчейна  

собираются  
структурируются  
и быстро ищутся  

## почему нельзя просто читать storage

читать storage напрямую дорого  
и неудобно  

особенно для истории действий  

## решение

контракты emit events  

а indexer слушает их  

## простой пример

пользователь сделал deposit  

контракт emit событие  

Deposit(user, amount)  

indexer увидел событие  
и сохранил данные  

## что такое хороший event design

Это правильное проектирование событий  

чтобы данные было удобно  

искать  
фильтровать  
и анализировать  

## что такое indexed

Solidity позволяет помечать параметры  

как indexed  

## зачем это нужно

indexed поля  

можно быстро искать  

## пример

indexed user  

теперь можно быстро найти  

все действия конкретного пользователя  

## важный момент

не всё нужно делать indexed  

есть ограничение  

## что обычно индексируют

user  
token  
id  

## частая ошибка

emit слишком мало данных  

или наоборот  

записывать лишнее  

## почему это важно для DeFi

analytics  
dashboard  
history  
reward tracking  

всё зависит от events  

## пример из жизни

events это как журнал операций  

indexer это система  

которая читает журнал  
и строит интерфейс  

## главная мысль

event design влияет  

на удобство indexing  

хорошие events делают систему  

понятной  
быстрой  
и удобной для фронтенда и аналитики
