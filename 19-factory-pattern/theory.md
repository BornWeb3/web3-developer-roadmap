# День 19: factory pattern и создание контрактов из контракта

## зачем это нужно

Обычно контракт деплоится один раз  

Но в реальных проектах нужно создавать много одинаковых контрактов  

Пример  

у каждого пользователя свой кошелек  
у каждого пула свой контракт  
у каждого проекта свой токен  

Деплоить всё вручную неудобно  

## что такое factory pattern

Factory это контракт  
который создаёт другие контракты  

Он как завод  

ты вызываешь функцию  
и он создаёт новый контракт  

## как это работает

есть factory контракт  

у него есть функция create  

когда ты её вызываешь  

создаётся новый контракт  
и возвращается его адрес  

## простой пример

пользователь нажимает кнопку  

factory создаёт новый контракт кошелька  

пользователь получает адрес  
и работает с ним  

## почему это удобно

всё создание происходит в одном месте  

можно хранить список всех созданных контрактов  

можно задавать параметры при создании  

## что происходит внутри

контракт использует специальную операцию создания  

EVM создаёт новый контракт  
и записывает его в блокчейн  

## передача параметров

при создании можно передать данные  

Пример  

создать токен с именем  
создать кошелек с владельцем  

## проблема обычного создания

адрес нового контракта заранее неизвестен  

он зависит от nonce  

## зачем это важно

иногда нужно знать адрес заранее  

для этого используют другой подход  

## пример из жизни

factory это как конвейер  

ты нажал кнопку  
и получил новый продукт  

## главная мысль

factory это контракт который создаёт другие контракты  

он упрощает управление  
делает систему масштабируемой  

если понимаешь factory  
ты начинаешь строить более сложные системы
